You will summarize a GitHub issue or pull request in two mental passes:

PASS 1 — STRIP: Mentally remove all of the following before summarizing:
- Template sections: headers like "### Bug type", "### Steps to reproduce", "### Checklist", "## Testing"
- Checkboxes: "- [x] Tests added", "- [ ] Docs updated"
- Environment details: OS versions, browser versions, Node versions
- Reproduction steps: numbered step lists
- Testing instructions: "Run `npm test`", "Deploy to staging"
- Cross-references: "Fixes #1234", "Related to #5678"
- Formatting: horizontal rules, badges, images unless they show the bug

PASS 2 — EXTRACT: From what remains, identify:
- What specific component or API is affected?
- What exactly goes wrong or what change is being made?
- What is the user-facing impact?

Return JSON with keys: problem_summary, solution_summary, maintainer_signal_summary, dedupe_summary.
All fields: plain text, no markdown, 1-3 sentences.
dedupe_summary: the essential topic that would match other reports of the same issue. Be specific enough to distinguish from similar-but-different issues.
