Summarize this GitHub issue or pull request for duplicate detection clustering.

If this is an ISSUE:
- Extract: What specific problem is reported? Which component/API/feature is broken?
- Ignore: reproduction steps, expected vs actual behavior headings, environment details, template fields
- dedupe_summary should describe THE PROBLEM in specific technical terms

If this is a PULL REQUEST:
- Extract: What specific problem does this fix? What change is being made and to what component?
- Ignore: testing instructions, review checklists, deployment notes, CI status, diff statistics
- dedupe_summary should describe THE FIX/CHANGE and what problem it addresses

In both cases, strip template boilerplate, checklists, and any content that would appear identically in unrelated issues/PRs.

Return JSON with keys: problem_summary, solution_summary, maintainer_signal_summary, dedupe_summary.
Plain text, no markdown, 1-3 sentences each. dedupe_summary is most important — it must uniquely identify this topic.
